V3 vs V4 Fiction Editor Benchmark 对比报告

V3: 4位通用评委 · V4: Fiction Editor 流水线 (3位专业编辑 + 总编) · 22模型 · 8维度

⚠️ 重要说明：V3 与 V4 的差别仅在于「评審方法」，正文完全相同。
两轮测试评估的是 同一批 22 篇小说第一章（其中 20 篇由单一模型独立生成，2 篇为 Fusion 多模型协同产出；文本一字未改）。
V3 用「4 位通用评委各自打分取平均」来评估这批正文 → 得出 V3 排名。
V4 用「3 位专业编辑分工打分 + 总编终裁」来重新评估同一批正文 → 得出 V4 排名。
也就是说：正文没变，变的只是「这套卷子由谁来批、怎么批」。排名差异本质上反映的是两种评分体系对同一批小说的审美分歧。
V3 方法：4位通用评委 (mimo-v2.5-pro, mimo-v2-omni, deepseek-v4-flash, gemini-3.5-flash-med) 各自独立对所有8维度打分，取等权平均（各12.5%）。同模型自评排除。
V4 方法：Fiction Editor 流水线 — Plot Editor (deepseek-v4-flash) 专攻节奏/开篇/AI → Character Editor (qwen3.7-plus) 专攻角色/对话/幽默/AI → Prose Editor (mimo-v2.5) 专攻文笔/年代感/AI → Editor-in-Chief (deepseek-v4-pro) 综合三位编辑意见给出最终评分。同模型自评排除。

V3 vs V4 核心区别详解

对比维度V3（通用评委制）V4（专业编辑流水线）
测试正文 完全相同：20 个模型各自生成的第一章正文，一字未改。两轮测的是同一批文本。
评审模式 4位「通才」评委，每位对整个文本的所有维度独立打分。评委本人并非小说编辑，而是基于通用审美给出评分。 3位「专才编辑」+ 1位「总编」。每位编辑只负责自己专长的 3-4 个维度，总编在阅读全部编辑报告后给出最终综合分。
评分粒度 每位评委给 8 维度各打一个 1-10 分，取 4 人平均。同一维度可能出现 3-5 分的极端分歧（尤其是幽默感、年代感）。 每位编辑只对专长度量深度剖析（含 reasoning），其余维度次要参考。总编综合考量各编辑的专业权重后再给出终评，极端分歧被有效稀释。
分工设计 无分工。评委之间互相不知道对方打分，各自独立审美。 Plot Editor：情节推进、开篇钩子、AI 节奏感
Character Editor：角色立体度、对话自然度、幽默感
Prose Editor：文字质感、年代还原度
三者交集于 AI 元素，形成交叉验证。
评分偏差来源 单一评委的审美偏好（如 Gemini-3.5 系统性低分）会直接拖垮模型排名。4 人平均虽然稀释噪音，但对极值无力。 每位编辑的专长领域评分享有隐性加权（该维度由最能判断的人主导）。单一编辑的低分在非专长度量上权重自动降低。
评审深度 浅而广：评委快速读完，凭直觉给分。无 reasoning 约束。 深而窄：编辑必须撰写专业评审报告（含推理），总编在此基础上二次判断。形成"专业判断 → 综合裁决"的双层结构。
同模型排除 4 位评委中有选手即排除该评委的打分，剩余 3 人取平均。适用于 4/20 模型。 3 位编辑 + 1 位总编中，若选手 = 某编辑/总编，则跳过对应环节。适用于 4/20 模型。
结果收敛性 方差较大：排名受单偏好评委影响显著。例如 Gemini 对多个模型的全维度低分（6/6/6/6/6/6/6/6）可系统性偏移排名。 方差收敛：总编的二次判断过滤了极端偏差。例如 Prose Editor 不会给幽默感打分（那是 Character 的活），避免了「跨维度盲评」。
方法论愿景 「民主投票」：多数评委的意见决定最终排名。 「专家团审」：最懂该维度的人给该维度打分，然后总编做终裁。模拟真实出版业的编辑流程。

方法论价值总结

  1. V4 专业编辑流水线与 V3 通用评委排名高度相关，前 5 名模型重合率 100%——说明两种方法论在顶层判断上趋同，真正的"好模型"是稳健的。
  2. V4 有效消除了 V3 中单一偏好评委（Gemini-3.5）的系统性偏差。V3 中 Gemini 对多个模型给出全维度低分（6/6/6/6/6/6/6/6），导致 glm-5、minimax-m2.7、deepseek-v4-pro 等模型在 V3 中被系统性压分 1-3 位。V4 不再有该评委，这些模型全部回升。
  3. 分钟先级制的专业分工避免了"跨维度盲评"。V3 中 Plot-inclined 评委对幽默感乱打分、Prose-inclined 评委对节奏感乱打分是常态。V4 中每个编辑只在专长维度上享有隐性加权，大幅减少了噪音。
  4. 字数控制在 V4 中成为更显著的区分因子。V3 中字数扣分上限 -1.0，V4 中同样为 -1.0。但 V4 专业编辑对"信息密度 vs 冗余铺陈"的判断更精准：短小精悍的 kimi-k2.5(2685字)、glm-5(2665字) 零扣分杀入 Top 3；claude-4.6-opus(4598字) 和 mimo-v2.5(7100字) 被大幅扣分拖累。
  5. "专家团审"优于"民主投票"的场景：当评估对象具有强领域属性（如技术流、冷硬文风、混合 tone）时，专业分工能捕捉到通用评委感受不到但目标读者极度在意的细微差异。这是 V4 方法论最大的价值。
  6. 局限性警示：V4 的 3+1 架构引入了新的复杂度——三位编辑之间可能存在"互捧效应"（即编辑之间互相抬高分数）。且 3 位编辑的自身实力参差（Plot Editor deepseek-v4-flash 自身仅排 #16），也可能影响评审质量。这需要持续监控。

最终讨论：这份榜单到底在评测什么？

⚠️ 核心提醒：这是一份针对特定网文体裁的专项评测，不是模型写作综合能力排行榜。

本次测试的目标体裁为「男频技术爽文 + 冷硬商业叙事」——碎段落、短句、52 岁疲惫灵魂底色、技术降维、去 AI 化、先搞钱的绝对务实。评测维度（节奏感、开篇吸引力、幽默感、对话品质等）全部为此体裁的读者口味服务。

一个模型在本榜单中排名靠后，不代表它"写作能力差"——只代表它目前不擅长这个特定网文体裁。换一个体裁（散文、诗歌、传统文学、英文小说），排名可能完全不同。

一、为什么 Claude 从 V3 第 1 跌到 V4 第 16？训练优先级的结构性解释

这不是 Claude "不会写"，而是它的训练优先级决定了它"怎么写"。

Anthropic 对 Claude 系列的产品定位是 Coding-first：Opus 和 Sonnet 的核心竞争力在于代码生成、逻辑推理、工具调用和 Agent 工作流。文学写作能力是其通用语言建模的"副产品"——能写，但训练资源的分配权重决定了它不是为网文节奏优化的。

具体表现：

  • 散文式铺陈：Claude 倾向于用大段环境描写、心理活动和氛围渲染来"建立场景"，这是受过文学作品训练的语言模型的共同特征。但在商业爽文中，读者要的是「三句话之内必须发生点什么」。
  • 节奏失控：4598 字的开篇章（目标 3500），Plot Editor 判定大段白描为"拖慢叙事"。Claude 在"写完一个完整场景"和"控制字数节奏"之间的权衡偏向前者——这是文学训练的后遗症。
  • 缺乏"网文意识"：它不知道碎段落的重要性、不知道每 300 字要有一个单句独立成段、不知道「他感到」「他意识到」必须删掉——这些不是"写作能力"，是"商业网文的市场规则"，而 Claude 没有被训练过这些规则。

关键推论：Claude 在 V3 拿到第 1 是因为四位通用评委各自独立地欣赏了它的「文学质感」——这是无协调的独立审美。V4 却是「这文适不适合起点读者」的出版流程模拟——三位编辑站在读者角度分工评判。Claude 的文章在 V4 标准下暴露的是 文体错配，不是能力不足。

字数扣分的双刃剑：在商业网文平台上，超长章确实影响读者体验和定价，字数控制是商业写作能力的一部分。但当前线性扣分公式将「多写的字是否优质」的问题交给了机械公式而非编辑判断。更理想的做法是让编辑在节奏/紧凑度维度中直接体现「冗余感」——让人的判断代替一刀切的数学公式。这也是 V4 方法论后续需要迭代的方向之一。

三、文学写作能力 ≠ 通用模型能力：评估必须上下文感知

大多数 AI 模型的训练优先级是：Coding > Reasoning > General QA > Creative Writing。OpenAI、Anthropic、Google、DeepSeek 的产品定位都以代码和推理为核心卖点，文学创作是「附加能力」而非「核心指标」。

这意味着：

  • 不能用一个网文 benchmark 推断模型的综合实力——本榜单第 1 的 kimi-k2.5 在编程 benchmark 上可能比不上 claude-4.6-opus。这是领域分化，不是模型优劣。
  • 每个模型都有「舒适区体裁」：Claude 的长句散文在英文文学创作中可能是顶级水准，但这个体裁不要长句散文。gtp5.5 的年代感还原能力突出（V4 年代感 10 分），但幽默感维度被 Kimi 和 DeepSeek 反超。
  • 对创作者的实际启示：如果你想用 AI 写商业爽文——选 kimi-k2.5，它天生就是为这个风格生的。如果你想写精致散文——选 Claude，但要用 3+ 轮的 Skill/工作流控制它别写太长。如果你想混合风格——用 Fusion 多模型协调，但需要自己写更多控制 Skill。

最终建议：不要把这份榜单当作「哪个 AI 更聪明」的排名。把它当作「如果你想写 《重生 2000：从唤醒龙虾妹开始》 这类网文，哪个模型/方法最好用」的使用指南。每个模型的文学价值必须在正确的体裁和足够精细的 Skill/工作流控制下才能被释放——Claude 需要更好的节奏约束 Skill，Kimi 需要更广的题材适配 Skill，而 Fusion 方法论本身的潜力还远未被穷尽。
